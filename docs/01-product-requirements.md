# 产品需求

## 定位

面向个人 NAS 的抖音媒体同步工具。

核心价值：

```text
个人账号资源同步
-> 下载视频或抽取原音频
-> 本地媒体库管理
-> 后续上传 WebDAV
```

## 目标用户

- 有个人 NAS 或家庭服务器。
- 使用抖音点赞、收藏保存音乐类视频。
- 希望在手机、车机、音乐软件中播放这些内容。
- 接受 Cookie 登录态方式，不追求全自动扫码登录。

## MVP 功能

### 1. 登录

- 单管理员密码。
- Session Cookie 登录态。
- 支持退出登录。
- 首次启动必须配置 `ADMIN_PASSWORD`。

### 2. 抖音账号管理

- 新增账号。
- 编辑账号名称。
- 粘贴 Cookie。
- 校验 Cookie 是否可用。
- 查看账号状态。
- 删除账号。

账号状态：

```text
active
expired
invalid
disabled
```

### 3. 视频资源同步

第一版支持：

- 点赞视频。
- 收藏视频。

后续扩展：

- 指定博主作品。
- 合集。
- 收藏夹。
- 短剧。

同步结果需要保存：

- 抖音视频 ID。
- 标题。
- 作者。
- 封面。
- 发布时间。
- 时长。
- 来源账号。
- 来源类型。
- 是否已下载。

### 4. 下载

下载类型：

```text
video
audio
both
```

视频：

- 优先保存原始视频流。
- 文件名避免非法字符。
- 保存来源元数据。

音频：

- 优先从视频中直接抽取原音轨。
- 不主动重编码。
- 根据实际编码选择扩展名。

推荐规则：

| 音频编码 | 输出 |
|---|---|
| aac | `.m4a` |
| alac | `.m4a` |
| mp3 | `.mp3` |
| opus | `.opus` 或 `.ogg` |
| unknown | 保留原容器或降级提示 |

### 5. 任务管理

- 下载队列。
- 任务状态。
- 失败原因。
- 手动重试。
- 手动取消。

任务状态：

```text
pending
running
succeeded
failed
canceled
```

### 6. 媒体文件管理

- 视频列表。
- 音频列表。
- 搜索。
- 按账号、作者、类型筛选。
- 删除文件。
- 打开详情。
- 查看来源视频。

## 后续功能

- WebDAV 自动上传。
- 定时同步。
- 音频封面写入。
- 标题清洗规则。
- 重复文件检测。
- PWA 安装到手机桌面。
- 移动端播放器预览。

## 非功能需求

- NAS 低资源可运行。
- Docker 一键部署。
- 数据和下载文件通过 volume 持久化。
- 前后端接口清晰，方便后续拆分 worker。
- 前端移动端优先，桌面端增强。
